기본적으로 Next.js는 모든 페이지를 **사전 렌더링**합니다. 이는 Next.js가 모든 페이지의 HTML을 클라이언트 사이드 JavaScript로 모두 처리하는 대신 미리 생성한다는 것을 의미합니다. 사전 렌더링은 성능과 SEO에 더 좋은 결과를 가져올 수 있습니다.

각 생성된 HTML은 해당 페이지에 필요한 최소한의 JavaScript 코드와 연결됩니다. 페이지가 브라우저에 의해 로드되면 JavaScript 코드가 실행되어 페이지를 완전히 상호 작용 가능하게 만듭니다(이 과정은 React에서 [hydration](https://react.dev/reference/react-dom/client/hydrateRoot)라고 합니다).

### 사전 렌더링

Next.js에는 **정적 생성**과 **서버 사이드 렌더링** 두 가지 형태의 사전 렌더링이 있습니다. 차이점은 페이지의 HTML을 생성하는 **시점**에 있습니다.

- 정적 생성: HTML이 **빌드 시** 생성되고 각 요청에 재사용됩니다.
- 서버 사이드 렌더링: HTML이 **각 요청**에 대해 생성됩니다.

중요한 점은 Next.js가 각 페이지에 대해 어떤 사전 렌더링 형태를 사용할지 선택할 수 있다는 것입니다. 정적 생성을 대부분의 페이지에 사용하고 다른 페이지에 대해 서버 사이드 렌더링을 사용하여 "하이브리드" Next.js 앱을 만들 수 있습니다.

성능상의 이유로 정적 생성을 서버 사이드 렌더링보다 권장합니다. 정적 생성된 페이지는 CDN에 의해 캐시될 수 있으며 성능을 향상시키기 위해 추가 구성이 필요하지 않습니다. 그러나 경우에 따라서는 서버 사이드 렌더링이 유일한 옵션일 수 있습니다.

또한 정적 생성 또는 서버 사이드 렌더링과 함께 클라이언트 사이드 데이터 가져오기를 사용할 수 있습니다. 이는 페이지의 일부를 클라이언트 사이드 JavaScript로 완전히 렌더링할 수 있음을 의미합니다. 자세한 내용은 [데이터 가져오기](/docs/pages/building-your-application/data-fetching/client-side) 문서를 참조하세요.
